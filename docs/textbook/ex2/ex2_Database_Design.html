自分でデータを収集し、テーブルのレイアウトを考える。

  • ２つ以上のテーブルを外部キーで結合できるようにする。
  • 他の人のテーブルをコピーしたりしないこと。
  • このデータベースを用いて、次回以降作成するWeb検索インターフェイスを作成することになる。
  • 外部に公開する予定がないならば、本講義に限っては著作権は気にせずに自由に作ってもらって構いません
  • モラルに欠けるような内容は避けてください。人前でプレゼンしても構わないぐらいの内容までに

 

演習２の手順：
  1. データベースの概念設計をE-R図を描くなどして行う。
  2. データベースに入れるテーブルとその内容（属性）、テーブル間の関係をおおまかに考える。
  3. テーブルの一次案を作成し、冗長属性の有無、表間の整合性のチェックを行い、正規化を行う。
  4. データベースの論理構成とレコード例をデモのデータベースのExcelファイルにならって作成する。Excelファイル内に注意書きを入れてますのでそれを守って下さい。
  5. 教員あるいはTAにチェックしてもらう。
※合格を得るまで先の演習には進めません。
 

※演習２では現段階でのデータベースの構想でOKです。演習２に合格後にデータベースに修正を入れても構いません。むしろ積極的に変更を加えてみて下さい。演習５で最終的な完成版のデータベースを改めて提出してもらうことになってますので、演習2に一旦合格すれば再提出してもらう必要はありません。ただし、題材から変更をしたいなど大幅な変更をする場合は相談下さい。
