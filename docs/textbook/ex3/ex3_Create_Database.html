(1) 自分が設計したデータベースについてSQLファイルを作成する。ファイル名やデータベース名(demo)、テーブル名は各自で名前を付けて下さい。

※demo.sqlはmoodleに置いているので参考にしてください。

  • 一行目の「create database demo」で指定したdemoのところがデータベース名になります。各自好きなデータベース名を付けて下さい。今後何度も出てくるので要確認。
  • テーブル名や属性名（列名）はすべて英数字小文字で記載してください。
  • テーブル名や属性名（列名）にバッククォート（`）を付けると予約語なども使用できるようになる。
    • 演習1のsetting.sqlはテーブル名や属性名にバッククォート（`）付けています。
    • MySQLの予約語とは予めMySQLに使われている関数の名前などに当たります。
    • 自分の使いたい属性名にエラーが出るなら（`）を加えると言う方策でいいと思います。
    • ただ、基本的には予約語の使用は避けて下さい。エラーの原因になることもあります。特に、select, from, like, whereなどqueryによく使う単語などは、SQL文で命令するときに属性名に毎回（`）が必要となり面倒です。

 

※テーブルの入力が面倒なので、CSV（Excel）ファイルから直接テーブルをインポートしたい！！

→csvからテーブルをimportする方法

 

(2) 作成した自分のSQLファイルをMySQLにインポート

docker exec -it docker-mysql bash

 

sqlファイルをインポートする

mysql -u user -ppassword < demo.sql　#ファイル名(demo.sql)は各自つけたDBのファイル名です

 

(4) データベースの確認

mysql -u user -ppassword
mysql> show databases;   #データベースのリストが表示されるので、自分のデータベースが作成されているか確認

mysql> use demo;  #自分の作成したデータベース名

mysql> show tables;  #テーブルのリストが表示されるので、作成したテーブルすべてが含まれているか確認

mysql> show fields from PDB;  #データ型や主キー等が正しく定義されているか確認

mysql> select * from PDB; #テーブルに含まれる全レコードを確認

 

(5) データベース検索テスト

  • MySQLでデータベースの操作を試す。自作のデータベースを使って演習１でやったようにいろいろSQL文を考えて検索してみて下さい。最終的に作成するデータベースサイトを想定してSQL文を考えてみてください。
  • データベースの内容を増やしたり修正したりしたい場合はSQLファイルに記述して再度インポートするようにしてください。MySQL内でデータの追加もできますが、エクスポートしないと、SQLファイルには反映されません。
※逆に言うと、MySQLモードでデータをいじっても、作成したSQLファイル(demo.sql等) には影響しません。MySQLを操作していてデータベースがおかしくなったら一旦データベースをdropして入れ直せば元に戻ります。
