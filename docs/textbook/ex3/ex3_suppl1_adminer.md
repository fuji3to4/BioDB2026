# 演習3 補足1: Adminerの使い方

> **対象環境:** Docker Compose (PostgreSQL 17 + PHP/Apache + Next.js 15)
> リポジトリをクローンして `docker compose up` で起動できます。

Adminer (<https://www.adminer.org/>) はブラウザからデータベースを確認・操作できる Web ツールです。
psql コマンドが難しいと感じる場合や、データを視覚的に確認したいときに活用してください。

---

## 1. アクセス方法

Docker Compose を起動した状態でブラウザで以下の URL にアクセスします。

<http://localhost/adminer-5.4.2.php>

---

## 2. ログイン

以下の情報でログインしてください。

| 項目 | 値 |
| --- | --- |
| データベース種類 | PostgreSQL |
| サーバ | postgres |
| ユーザ名 | user |
| パスワード | password |
| データベース | （空欄のままログインできます） |

※「サーバ」は `localhost` ではなく `postgres`（Docker ネットワーク上のサービス名）を入力してください。

---

## 3. データベースの確認

ログイン後、左メニューにデータベース一覧が表示されます。

* 自分が作成したデータベース名をクリック → テーブル一覧が表示される
* テーブル名をクリック → 列定義（型・主キー・制約）を確認できる
* 「データを選択」→ テーブルの全レコードを一覧表示できる

psql コマンドとの対応：

| psql コマンド | Adminer での操作 |
| --- | --- |
| `\l`（DB 一覧） | ログイン画面の DB 一覧 または 左メニュー |
| `\dt`（テーブル一覧） | DB を選択 → テーブル一覧が表示 |
| `\d テーブル名`（列定義） | テーブル名をクリック → 「テーブル構造」 |
| `select * from テーブル名;` | テーブルの「データを選択」 |

---

## 4. SQL 文の実行

上部メニューの「SQL コマンド」をクリックすると、SQL を直接入力して実行できます。

```sql
SELECT * FROM pdb WHERE resolution < 2
```

psql の対話モードと同様に自由な SQL 文を試すことができます。結果は表形式で表示されます。

→ 演習3の「データベース検索テスト」は Adminer の「SQL コマンド」でも実行できます。

---

## 5. CSV からのデータインポート

Excel で作成したデータを CSV として保存しておくと、Adminer からも直接テーブルにインポートできます。

1. Adminer で対象のデータベースを選択
2. 左メニューの「インポート」をクリック
3. CSV ファイルを選択してアップロード
4. 「インポート」ボタンを押す

注意：CSV の列の順番とテーブルの列定義が一致している必要があります。
事前に `CREATE TABLE` でテーブルを作成しておいてください。

---

## 6. 注意事項

* Adminer での操作（レコード追加・削除・編集）は **SQL ファイルには自動保存されません**。
  変更を残したい場合は必ず SQL ファイルも更新してください。
* Docker Compose が起動していないと Adminer にアクセスできません。
