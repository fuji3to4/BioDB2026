# 演習3: データベース作成

> **対象環境:** Docker Compose (PostgreSQL 17 + PHP/Apache + Next.js 15)
> リポジトリをクローンして `docker compose up` で起動できます。

(1) 自分が設計したデータベースについてSQLファイルを作成する。**ファイル名やデータベース名(demo)、テーブル名は各自で名前を付けて下さい。**

※[demo.sql](../../SQL/demo.sql) はリポジトリの `SQL/` フォルダに置いているので参考にしてください。

* 一行目の**「create database demo」**で指定したdemoのところが**データベース名**になります。**各自好きなデータベース名**を付けて下さい。今後何度も出てくるので要確認。
* テーブル名や属性名（列名）は**すべて英数字小文字（+アンダースコア"\_"）**で記載してください。
* テーブル名や属性名（列名）に**ダブルクォート（"）を付けると予約語なども使用できるようになる**ですが、基本的には予約語の使用は避けて下さい。
  * 演習1の [setting.sql](../../SQL/) はテーブル名や属性名にダブルクォート（"）付けています。
  * PostgreSQLの予約語とは予めPostgreSQLに使われている関数の名前などに当たります。
  * 自分の使いたい属性名にエラーが出るなら（"）を加えると言う方策でいいと思います。
  * ただ、基本的には予約語の使用は避けて下さい。エラーの原因になることもあります。特に、select, from, like, whereなどqueryによく使う単語などは、SQL文で命令するときに属性名に毎回（"）が必要となり面倒です。

※テーブルの入力が面倒なので、CSV（Excel）ファイルから直接テーブルをインポートしたい！！

→csvからテーブルをimportする方法

(2) 作成した自分のSQLファイルをPostgreSQLにインポート

以下のコマンドを実行します（ファイル名は各自作成したSQLファイル名に置き換えてください）。

```bash
docker compose exec postgres psql -U user -d postgres -f /home/user/SQL/demo.sql
```

※ファイル名（`demo.sql`）は各自つけたDBのファイル名です。`./SQL/` フォルダに配置されたSQLファイルがコンテナ内の `/home/user/SQL/` に対応しています。

---

(4) データベースの確認

psqlに接続して確認します。

```bash
docker compose exec postgres psql -U user -d demo
```

```
demo=# \l
  #データベースのリストが表示されるので、自分のデータベースが作成されているか確認

demo=# \c demo   #自分の作成したデータベース名に切り替え

demo=# \dt
  #テーブルのリストが表示されるので、作成したテーブルすべてが含まれているか確認

demo=# \d PDB   #データ型や主キー等が正しく定義されているか確認

demo=# select * from PDB;  #テーブルに含まれる全レコードを確認
```

---

(5) データベース検索テスト

* psqlでデータベースの操作を試す。自作のデータベースを使って演習１でやったようにいろいろSQL文を考えて検索してみて下さい。最終的に作成するデータベースサイトを想定してSQL文を考えてみてください。
* データベースの内容を増やしたり修正したりしたい場合はSQLファイルに記述して再度インポートするようにしてください。psql内でデータの追加もできますが、エクスポートしないと、SQLファイルには反映されません。

※逆に言うと、psqlでデータをいじっても、作成したSQLファイル(demo.sql等)には影響しません。psqlを操作していてデータベースがおかしくなったら一旦データベースをdropして入れ直せば元に戻ります。
